## Project Overview

POC backend for an **annual planning system** for educational institutions. A coordinator can plan how knowledge/skills will be taught across subjects throughout the school year.

## Domain Concepts

### Educational Structure
```
areas (e.g., "Sciences", "Humanities")
  └── subjects (e.g., "Mathematics", "Physics", "History")
```
- **Areas** have a coordinator (user)
- **Subjects** belong to an area

### Student Organization
```
courses (e.g., "3a", "5b") - groups of students
  └── students
  └── course_subjects - instance of a course taking a subject with a teacher
```
- **Courses** = student groups that advance through the year (like "3rd Grade Section A")
- **Course Subjects** = a course taking a specific subject with a teacher during a school year

### Course Schedule (with Shared Classes)

Each course has a `schedule` JSONB field defining weekly class slots:

```json
{
  "monday": [
    {"time": "08:00-09:30", "subject": "Matemáticas"},
    {"time": "09:45-11:15", "subject": "Física"}
  ],
  "wednesday": [
    {"time": "08:00-09:30", "subject": "Física", "shared_with": "Matemáticas"},
    {"time": "09:45-11:15", "subject": "Biología"}
  ]
}
```

**Shared Classes**: When `shared_with` is present, two subjects are taught simultaneously by two teachers:
- The slot's `subject` is the primary subject
- `shared_with` indicates the secondary subject taught at the same time
- Both subjects must belong to the same area for the feature to apply

### Knowledge Taxonomy (what gets taught)
```
problematic_nuclei (high-level themes)
  └── knowledge_areas (specific knowledge domains)
        └── categories (granular skills/concepts)
```
Example:
- Nucleus: "Logical-Mathematical Thinking"
  - Knowledge Area: "Basic Arithmetic"
    - Category: "Addition and Subtraction"
    - Category: "Multiplication"

### Coordination Document (documento de coordenadas)

The **main output** of this system. A coordination document is a **planning document** that:

1. **Belongs to an Area** (includes all subjects of that area)
2. **Defines a time period** (start_date, end_date)
3. **Has a status** (draft, published, archived)
4. **For each subject specifies**:
   - Number of classes (`class_count`)
   - Which categories/concepts will be covered (`category_ids`)
   - Class-by-class plan (`class_plan`) - generated by LLM
5. **Includes methodological strategies** - LLM-generated text explaining HOW to teach

**`subjects_data` JSONB structure:**
```json
{
  "1": {
    "class_count": 20,
    "category_ids": [1, 2, 5],
    "class_plan": [
      {"class_number": 1, "title": "Introduction", "category_ids": [1]},
      {"class_number": 2, "title": "Deep dive", "category_ids": [1, 2]}
    ]
  }
}
```
Key = subject_id

## AI Integration

Uses **Azure OpenAI** (gpt-5-mini) for:

1. **Methodological Strategy Generation** - Creates pedagogical approach text
2. **Class Plan Generation** - Auto-generates titles and category assignments per class
3. **Chat Assistant (Alizia)** - Helps modify documents via natural language

### Chat Tools (Function Calling)

| Function | Description |
|----------|-------------|
| `update_document_title` | Changes document title |
| `update_methodological_strategy` | Replaces entire strategy |
| `append_to_methodological_strategy` | Adds text to end of strategy (preserves existing) |
| `update_class_title` | Changes a specific class title |
| `update_class_categories` | Updates categories for a class |

### Azure OpenAI Config

- Endpoint: `AZURE_OPENAI_ENDPOINT` env var
- API Key: `AZURE_OPENAI_API_KEY` env var
- Model: `gpt-5-mini` (requires `max_completion_tokens`, no custom temperature)

### Frontend Flow

1. **Login** → Select user (coordinator)
2. **Home** → Select a course (3a, 5b, etc.)
3. **Course Detail** → Two tabs:
   - "Sobre el curso" - students, details, schedule grid
   - "Doc. de coordenadas" - list of existing documents with actions
4. **Document List Features**:
   - Shows all documents for the area with status badges (Borrador/Publicado)
   - **Publicar** button (green) - marks document as published
   - **Editar** button - opens document view
   - **Borrar** button - deletes with confirmation
5. **Create Document Wizard** (3 steps):
   - Step 1: Select problematic nucleus and categories
   - Step 2: Set dates + classes per subject
   - Step 3: Assign categories to each subject (drag & drop)
6. **Document View** → Full editing interface:
   - Editable title (click to edit)
   - Methodological strategy (double-click to edit manually, or use AI)
   - "Generar con IA" button - generates strategy + class plans
   - Timeline display
   - Classes per discipline (expandable sections)
   - Per-class: editable title, category tags (removable), add category button (+)
   - **Unassigned categories warning** - shows concepts not yet assigned to any class
   - Chat with AI assistant (Alizia) with full conversation history

## Key Implementation Notes

### Chat History
- Frontend maintains `state.chatHistory` array
- Sends full history with each request (`{ history: [...] }`)
- Backend builds messages array from history for context continuity

### Category Management
- Document-level: `category_ids` array (selected in wizard)
- Class-level: each class in `class_plan` has its own `category_ids`
- Warning shown if document categories aren't all assigned to classes

### Document Status
- `draft` (default) - yellow badge
- `published` - green badge, "Publicar" button hidden
- `archived` - gray badge

### Shared Classes Feature

**Concept**: Two subjects taught simultaneously by two teachers in the same time slot.

**Data Model**:
- Schedule slot has optional `shared_with` field pointing to another subject name
- Example: `{"time": "08:00-09:30", "subject": "Física", "shared_with": "Matemáticas"}`

**Backend Helper Functions** (`main.py`):
- `get_shared_classes_for_area()`: Detects slots with shared_with where both subjects belong to the area
- `calculate_shared_class_numbers()`: Calculates which class numbers (1, 2, 3...) fall on shared time slots

**Frontend Visualization**:
1. **Schedule Grid** (coordinator & teacher views): Shows shared slots with 🤝 badge and both subject names
2. **Teacher Class List**: Individual class items show "🤝 Compartida" badge when that class falls on a shared slot

**Algorithm for Class Number Calculation**:
```
For Matemáticas in course 3a with schedule:
- Monday 08:00: Matemáticas → Clase 1
- Tuesday 09:45: Matemáticas → Clase 2
- Wednesday 08:00: Física (shared_with: Matemáticas) → Clase 3 ← SHARED
- Thursday 08:00: Matemáticas → Clase 4
- Friday 09:45: Matemáticas → Clase 5

Result: Class 3 is marked as shared
Pattern repeats: If 10 classes, then 3 and 8 are shared (every 5 classes)
```
