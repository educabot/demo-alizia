# T-4.3.1: Usecase actualizar secciones del documento

**Historia:** HU-4.3 — Secciones dinámicas
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Implementar el usecase `UpdateSections` que permite editar secciones individuales del documento. Valida que cada key existe en la configuración `coord_doc_sections` de la organización, que las secciones de tipo `select_text` incluyen `selected_option`, y que el documento está en estado `draft`. Hace merge parcial con el JSONB existente (no reemplaza todo).

## Implementación

Archivo: `src/core/usecases/coordination/update_sections.go`

```go
package coordination

import (
    "context"
    "encoding/json"
    "fmt"

    "go-alizia/src/core/providers"
)

type UpdateSectionsInput struct {
    DocumentID uint                              `json:"-"`
    Sections   map[string]map[string]interface{} `json:"sections"`
}

// Ejemplo de input.Sections:
// {
//   "problem_edge": { "value": "Nuevo contenido..." },
//   "methodological_strategy": { "selected_option": "proyecto", "value": "..." }
// }

type UpdateSectionsUsecase struct {
    coordProvider providers.CoordinationProvider
    orgProvider   providers.OrganizationProvider
}

func (uc *UpdateSectionsUsecase) Execute(ctx context.Context, orgID uint, input UpdateSectionsInput) error {
    // 1. Obtener el documento
    doc, err := uc.coordProvider.GetByID(ctx, orgID, input.DocumentID)
    if err != nil {
        return fmt.Errorf("documento no encontrado: %w", err)
    }

    // 2. Verificar que está en estado draft
    if doc.Status != "draft" {
        return fmt.Errorf("solo se pueden editar documentos en estado draft")
        // → handler mapea a 403
    }

    // 3. Obtener config de secciones de la org
    org, err := uc.orgProvider.GetByID(ctx, orgID)
    if err != nil {
        return err
    }
    sectionConfigs := org.Config.CoordDocSections // []SectionConfig

    // 4. Validar cada key del input
    for key, sectionData := range input.Sections {
        config := findSectionConfig(sectionConfigs, key)
        if config == nil {
            return fmt.Errorf("section_key '%s' no existe en la configuracion: validation_error", key)
            // → handler mapea a 422
        }

        // 5. Si es select_text, validar que tiene selected_option valido
        if config.Type == "select_text" {
            selectedOption, ok := sectionData["selected_option"].(string)
            if !ok || selectedOption == "" {
                return fmt.Errorf("section '%s' de tipo select_text requiere selected_option: validation_error", key)
                // → 422
            }
            if !contains(config.Options, selectedOption) {
                return fmt.Errorf("selected_option '%s' no es valida para section '%s': validation_error", selectedOption, key)
                // → 422
            }
        }
    }

    // 6. Merge parcial con sections existentes
    var existingSections map[string]interface{}
    json.Unmarshal(doc.Sections, &existingSections)
    if existingSections == nil {
        existingSections = make(map[string]interface{})
    }

    for key, value := range input.Sections {
        existingSections[key] = value
    }

    // 7. Serializar y actualizar
    updatedJSON, _ := json.Marshal(existingSections)
    doc.Sections = updatedJSON

    return uc.coordProvider.Update(ctx, doc)
}
```

## Notas

- El merge parcial es clave: si el documento tiene secciones `problem_edge` y `methodological_strategy`, y el PATCH solo envia `problem_edge`, la estrategia no se toca
- La validacion de `selected_option` verifica que el valor esta en la lista de `options` de la config
- El error de tipo `validation_error` se usa como convención para que el handler lo mapee a 422
- Solo documentos en `draft` son editables — documentos publicados o archivados retornan 403
- `findSectionConfig` es un helper que busca en el array de SectionConfig por key
- El campo `sections` del documento usa `datatypes.JSON` (GORM), que se puede unmarshal/marshal a `map[string]interface{}`
