# T-4.3.2: Usecase generar secciones con IA

**Historia:** HU-4.3 — Secciones dinámicas
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Implementar el usecase `GenerateSections` que genera contenido con Azure OpenAI para cada sección configurada en la organización. Lee el `ai_prompt` de cada sección, reemplaza placeholders como `{selected_option}`, y envía el prompt junto con el contexto del documento (topics, materias, nombre del area). Este usecase se llama desde el endpoint `/generate` que también dispara la generación del plan de clases (HU-4.4).

## Implementación

Archivo: `src/core/usecases/coordination/generate_sections.go`

```go
package coordination

import (
    "context"
    "encoding/json"
    "fmt"
    "strings"

    "go-alizia/src/core/providers"
)

type GenerateSectionsUsecase struct {
    coordProvider providers.CoordinationProvider
    orgProvider   providers.OrganizationProvider
    aiProvider    providers.AIProvider
}

func (uc *GenerateSectionsUsecase) Execute(ctx context.Context, orgID, docID uint) error {
    // 1. Obtener documento con preloads (topics, subjects)
    doc, err := uc.coordProvider.GetByID(ctx, orgID, docID)
    if err != nil {
        return err
    }

    // 2. Obtener config de la org
    org, err := uc.orgProvider.GetByID(ctx, orgID)
    if err != nil {
        return err
    }

    // 3. Construir contexto del documento para el prompt
    docContext := buildDocumentContext(doc)
    // Incluye: nombre del area, lista de topics seleccionados,
    // materias con sus topics, periodo

    // 4. Obtener secciones existentes (para leer selected_option)
    var existingSections map[string]map[string]interface{}
    json.Unmarshal(doc.Sections, &existingSections)

    // 5. Para cada seccion configurada, generar contenido
    generatedSections := make(map[string]interface{})

    for _, sectionConfig := range org.Config.CoordDocSections {
        prompt := sectionConfig.AIPrompt

        // Reemplazar placeholder {selected_option} si aplica
        if sectionConfig.Type == "select_text" {
            if existing, ok := existingSections[sectionConfig.Key]; ok {
                if opt, ok := existing["selected_option"].(string); ok {
                    prompt = strings.ReplaceAll(prompt, "{selected_option}", opt)
                }
            }
        }

        // Agregar contexto del documento al prompt
        fullPrompt := fmt.Sprintf("%s\n\nContexto del documento:\n%s", prompt, docContext)

        // Llamar a Azure OpenAI
        generated, err := uc.aiProvider.GenerateText(ctx, fullPrompt)
        if err != nil {
            return fmt.Errorf("error generando seccion '%s': %w", sectionConfig.Key, err)
        }

        // Construir el objeto de la seccion
        sectionData := map[string]interface{}{
            "value": generated,
        }
        // Preservar selected_option si existia
        if sectionConfig.Type == "select_text" {
            if existing, ok := existingSections[sectionConfig.Key]; ok {
                if opt, ok := existing["selected_option"]; ok {
                    sectionData["selected_option"] = opt
                }
            }
        }

        generatedSections[sectionConfig.Key] = sectionData
    }

    // 6. Guardar todas las secciones generadas
    sectionsJSON, _ := json.Marshal(generatedSections)
    doc.Sections = sectionsJSON

    return uc.coordProvider.Update(ctx, doc)
}

// buildDocumentContext construye un string con el contexto del documento
// para enriquecer el prompt enviado a la IA.
func buildDocumentContext(doc *entities.CoordinationDocument) string {
    // Ejemplo de output:
    // Area: Ciencias Exactas
    // Periodo: 2026-03-15 a 2026-07-15
    // Topics seleccionados: Pensamiento logico, Aritmetica basica, ...
    // Materias: Matematicas (20 clases), Fisica (18 clases)
    // Topics por materia:
    //   Matematicas: Pensamiento logico, Aritmetica basica
    //   Fisica: Mecanica clasica
}
```

## Notas

- Este usecase genera contenido para **todas** las secciones — no es selectivo
- El endpoint `/generate` llama tanto a este usecase como al de generación de plan de clases (HU-4.4) — ambos corren en la misma request
- Si una sección de tipo `select_text` no tiene `selected_option` definido, se omite el reemplazo del placeholder (o se puede retornar error)
- El `ai_prompt` viene de la config de la org — cada organización puede personalizar los prompts
- `buildDocumentContext` es clave para que la IA genere contenido relevante — incluye topics, materias y periodo
- Los errores de IA no deben ser silenciosos: si falla la generación de una sección, se retorna error al usuario
- Considerar implementar generación paralela de secciones (goroutines) si el tiempo de respuesta es un problema
