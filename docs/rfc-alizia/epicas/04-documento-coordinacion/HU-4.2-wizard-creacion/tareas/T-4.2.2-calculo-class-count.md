# T-4.2.2: Cálculo automático de class_count

**Historia:** HU-4.2 — Wizard de creación
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Implementar una función helper que calcula el `class_count` sugerido para una materia dentro de un período. La lógica cuenta cuántos `time_slot_subjects` tiene un `course_subject` y lo multiplica por las semanas del período. Este valor es una sugerencia que el coordinador puede modificar manualmente (por feriados, eventos, etc.).

## Implementación

Archivo: `src/core/usecases/coordination/calculate_class_count.go`

```go
package coordination

import (
```

```
semanas = floor((end_date - start_date) / 7 dias)
class_count = slots_por_semana × semanas
```

## Notas

- Este calculo es una **sugerencia**, no un valor definitivo — el coordinador puede hacer override en el paso 2 del wizard
- Los slots se cuentan desde `time_slot_subjects` asociados a `course_subjects` del area
- No se descuentan feriados ni semanas de examen — eso queda a criterio del coordinador
- Se usa `math.Floor` para redondear hacia abajo (mejor que sobren clases a que falten)
- Esta funcion se puede exponer como endpoint auxiliar (`GET /api/v1/areas/:id/suggested-class-counts?start_date=...&end_date=...`) o llamarse desde el frontend al seleccionar el periodo
- Si una materia no tiene time_slots asignados, su class_count sugerido es 0
