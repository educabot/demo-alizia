# T-4.1.3: Provider interface para CoordinationDocument

**Historia:** HU-4.1 — Modelo de datos del documento
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Definir la interfaz `CoordinationProvider` con todos los métodos necesarios para CRUD y operaciones complejas del documento de coordinación. Todas las operaciones deben filtrar por `organization_id` para respetar multi-tenancy.

## Implementación

Archivo: `src/core/providers/coordination_provider.go`

```go
package providers

import (
    "context"

    "go-alizia/src/core/entities"
)

type CoordinationProvider interface {
    // CRUD principal
    Create(ctx context.Context, doc *entities.CoordinationDocument) error
    GetByID(ctx context.Context, orgID, docID uint) (*entities.CoordinationDocument, error)
    List(ctx context.Context, orgID uint, areaID *uint) ([]entities.CoordinationDocument, error)
    Update(ctx context.Context, doc *entities.CoordinationDocument) error
    Delete(ctx context.Context, orgID, docID uint) error

    // Topics a nivel documento
    AddTopics(ctx context.Context, docID uint, topicIDs []uint) error

    // Subjects del documento
    SetSubjects(ctx context.Context, docID uint, subjects []entities.CoordinationDocumentSubject) error

    // Topics por materia
    AddSubjectTopics(ctx context.Context, subjectDocID uint, topicIDs []uint) error

    // Clases
    CreateClasses(ctx context.Context, subjectDocID uint, classes []entities.CoordDocClass) error
    UpdateClass(ctx context.Context, class *entities.CoordDocClass) error
}
```

## Notas

- `GetByID` recibe `orgID` para validar que el documento pertenece a la org (multi-tenancy)
- `List` recibe `areaID *uint` como filtro opcional — si es nil, retorna todos los docs de la org
- `SetSubjects` reemplaza todas las materias (no hace append); se usa en la creación del wizard
- `AddTopics` y `AddSubjectTopics` son operaciones de append — no eliminan topics existentes
- `CreateClasses` crea varias clases de golpe (generadas por IA o wizard)
- `UpdateClass` actualiza una clase individual (title, objective, topics)
- No se incluye `UpdateSubject` separado porque el class_count se define en creación; si se necesita luego, se agrega
- El contexto (`context.Context`) se usa para propagación de timeouts y cancelación
