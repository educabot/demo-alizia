# T-3.1.3: Repository GORM de Organization

**Historia:** HU-3.1 — Organizaciones y configuración
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Implementar el repository GORM que satisface la interface `OrganizationProvider`. El método `GetByID` filtra por org_id (multi-tenancy). El método `Update` realiza un merge parcial del JSONB de config usando `gorm.Expr` para que no se pierdan keys existentes.

## Implementación

### `src/repositories/organization.go`

```go
package repositories

import (
    "context"
    "encoding/json"
    "errors"

    "gorm.io/gorm"

    "github.com/educabot/demo-alizia/src/core/entities"
    "github.com/educabot/demo-alizia/src/core/providers"
    apperrors "github.com/team-ai-toolkit/errors"
)

type organizationRepo struct {
    db *gorm.DB
}

func NewOrganizationRepository(db *gorm.DB) providers.OrganizationProvider {
    return &organizationRepo{db: db}
}

func (r *organizationRepo) GetByID(ctx context.Context, id uint) (*entities.Organization, error) {
    var org entities.Organization
    if err := r.db.WithContext(ctx).First(&org, id).Error; err != nil {
        if errors.Is(err, gorm.ErrRecordNotFound) {
            return nil, apperrors.ErrNotFound
        }
        return nil, err
    }
    return &org, nil
}

func (r *organizationRepo) Update(ctx context.Context, id uint, updates map[string]interface{}) (*entities.Organization, error) {
    // Si el update incluye "config", hacer merge JSONB en lugar de reemplazar
    if configUpdate, ok := updates["config"]; ok {
        configJSON, err := json.Marshal(configUpdate)
        if err != nil {
            return nil, err
        }

        // JSONB merge: config || newConfig (las keys nuevas se agregan, las existentes se sobreescriben)
        err = r.db.WithContext(ctx).
            Model(&entities.Organization{}).
            Where("id = ?", id).
            Update("config", gorm.Expr("config || ?::jsonb", string(configJSON))).
            Error
        if err != nil {
            return nil, err
        }

        // Remover config del map para no hacer doble update
        delete(updates, "config")
    }

    // Actualizar el resto de campos (name, slug, etc.) si los hay
    if len(updates) > 0 {
        err := r.db.WithContext(ctx).
            Model(&entities.Organization{}).
            Where("id = ?", id).
            Updates(updates).
            Error
        if err != nil {
            return nil, err
        }
    }

    // Retornar la org actualizada
    return r.GetByID(ctx, id)
}
```

## Notas

- El operador `||` de PostgreSQL hace merge de JSONB: las keys del segundo operando sobreescriben las del primero, pero las keys no mencionadas se conservan.
- Se usa `gorm.Expr("config || ?::jsonb", ...)` para que GORM no escape el valor como string simple.
- Siempre filtrar por `id` que venga validado contra el `org_id` del JWT (multi-tenancy se valida en el handler/middleware).
- Mapear `gorm.ErrRecordNotFound` a `apperrors.ErrNotFound` para respuestas HTTP consistentes.
- El merge JSONB es de un solo nivel de profundidad. Para nested objects, PostgreSQL `||` reemplaza el objeto completo del key. Si se necesita deep merge, hay que usar `jsonb_deep_merge` o hacerlo en Go.
