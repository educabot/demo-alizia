## Descripcion

Implementar el repository GORM para teacher_lesson_plans con preloads de actividades y relaciones.

## Implementacion

```go
// src/repositories/teacher_lesson_plan_repository.go

type TeacherLessonPlanRepository struct {
    db *gorm.DB
}

func (r *TeacherLessonPlanRepository) GetByID(id uint) (*entities.TeacherLessonPlan, error) {
    var plan entities.TeacherLessonPlan
    err := r.db.
        Preload("Activities", func(db *gorm.DB) *gorm.DB {
            return db.Order(`"order" ASC`).Preload("Activity")
        }).
        Preload("CoordDocClass").
        First(&plan, id).Error
    return &plan, err
}

func (r *TeacherLessonPlanRepository) GetByCourseSubject(courseSubjectID uint) ([]entities.TeacherLessonPlan, error) {
    var plans []entities.TeacherLessonPlan
    err := r.db.
        Where("course_subject_id = ?", courseSubjectID).
        Preload("Activities", func(db *gorm.DB) *gorm.DB {
            return db.Order(`"order" ASC`).Preload("Activity")
        }).
        Order("class_number ASC").
        Find(&plans).Error
    return plans, err
}
```

## Notas

- El preload de Activities incluye orden ASC y nested preload de Activity (para tener nombre/descripcion)
- `"order"` va entre comillas porque es palabra reservada en SQL
