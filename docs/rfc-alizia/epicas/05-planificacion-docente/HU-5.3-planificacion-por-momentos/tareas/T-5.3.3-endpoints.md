# T-5.3.3: Endpoints POST y PATCH

**Historia:** HU-5.3 — Planificación por momentos
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Endpoints para crear y editar planificaciones docentes.

## Implementación

### Endpoints

```
POST  /api/v1/lesson-plans        → Crear planificación
PATCH /api/v1/lesson-plans/:id    → Editar planificación
GET   /api/v1/lesson-plans/:id    → Detalle de una planificación
```

### Handler

```go
func (h *LessonPlanHandler) Create(c *gin.Context) {
    var input usecases.CreateLessonPlanInput
    if err := c.ShouldBindJSON(&input); err != nil {
        c.JSON(400, gin.H{"error": err.Error()})
        return
    }

    user := auth.GetUser(c)
    plan, err := h.createUsecase.Execute(user.ID, input)
    if err != nil {
        handleError(c, err)
        return
    }
    c.JSON(201, plan)
}

func (h *LessonPlanHandler) Update(c *gin.Context) {
    planID := parseID(c, "id")
    var input usecases.UpdateLessonPlanInput
    if err := c.ShouldBindJSON(&input); err != nil {
        c.JSON(400, gin.H{"error": err.Error()})
        return
    }

    user := auth.GetUser(c)
    plan, err := h.updateUsecase.Execute(user.ID, planID, input)
    if err != nil {
        handleError(c, err)
        return
    }
    c.JSON(200, plan)
}

func (h *LessonPlanHandler) GetByID(c *gin.Context) {
    planID := parseID(c, "id")
    plan, err := h.lessonPlanProvider.GetByID(planID)
    if err != nil {
        handleError(c, err)
        return
    }
    c.JSON(200, plan)
}
```

## Notas

- POST retorna 201, PATCH retorna 200
- GET detalle incluye activities con preload de activity (nombre, descripción)
