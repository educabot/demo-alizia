# T-5.5.1: Usecase — cambiar estado

**Historia:** HU-5.5 — Estados de planificación
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Usecase para cambiar el estado de una planificación entre pending y planned, con validaciones.

## Implementación

```go
// src/core/usecases/change_lesson_plan_status.go

type ChangeLessonPlanStatusInput struct {
    Status entities.LessonPlanStatus `json:"status" validate:"required,oneof=pending planned"`
}

func (uc *ChangeLessonPlanStatusUsecase) Execute(userID uint, planID uint, input ChangeLessonPlanStatusInput) (*entities.TeacherLessonPlan, error) {
    plan, _ := uc.lessonPlanProvider.GetByID(planID)

    // Auth
    cs, _ := uc.courseSubjectProvider.GetByID(plan.CourseSubjectID)
    if cs.TeacherID != userID {
        return nil, errors.NewForbidden("only assigned teacher")
    }

    // Validar transición a planned
    if input.Status == entities.LessonPlanStatusPlanned {
        if len(plan.Activities) == 0 {
            return nil, errors.NewValidation("plan must have activities")
        }
        if plan.Proposal == nil || *plan.Proposal == "" {
            return nil, errors.NewValidation("plan must have a generated proposal")
        }
    }

    plan.Status = input.Status
    uc.lessonPlanProvider.Update(plan)

    return plan, nil
}
```

## Notas

- La transición pending → planned requiere actividades + proposal
- La transición planned → pending no tiene restricciones (permite re-edición)
