## Descripcion

Endpoint para que el docente vea su cronograma de clases con estado de planificacion.

## Implementacion

### Endpoint

```
GET /api/v1/course-subjects/:id/lesson-plans
```

### Handler

```go
func (h *LessonPlanHandler) GetTeacherSchedule(c *gin.Context) {
    courseSubjectID := parseID(c, "id")

    // Auth: verificar que el usuario es el docente asignado o coordinador del area
    user := auth.GetUser(c)
    cs, _ := h.courseSubjectProvider.GetByID(courseSubjectID)
    if user.ID != cs.TeacherID && !isCoordinatorOfArea(user, cs.Subject.AreaID) {
        c.JSON(403, gin.H{"error": "not authorized"})
        return
    }

    result, err := h.getScheduleUsecase.Execute(courseSubjectID)
    if err != nil {
        handleError(c, err)
        return
    }
    c.JSON(200, result)
}
```

### Validaciones

- `course_subject_id` debe existir → 404
- Usuario debe ser docente del course_subject o coordinador del area → 403
- Debe existir un documento publicado para el area → 404
