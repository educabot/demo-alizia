# T-2.2.1: Migración — columna profile_data

**Historia:** HU-2.2 — Formulario de perfil
**Tipo:** DB
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripción

Agregar columna `profile_data` JSONB a la tabla `users` para almacenar los datos de perfil dinámicos capturados durante el onboarding.

## Migración

### `migrations/XXX_add_profile_data_to_users.up.sql`

```sql
ALTER TABLE users ADD COLUMN profile_data JSONB NOT NULL DEFAULT '{}';

COMMENT ON COLUMN users.profile_data IS 'Dynamic profile fields captured during onboarding. Schema defined by organization config.';
```

### `migrations/XXX_add_profile_data_to_users.down.sql`

```sql
ALTER TABLE users DROP COLUMN profile_data;
```

## Ejemplo de datos almacenados

```json
{
  "disciplines": ["Matemáticas", "Física"],
  "years_of_experience": 12,
  "specialization": "Didáctica de las ciencias",
  "education_levels": ["Secundario", "Superior"]
}
```

## Notas

- Los keys del JSON corresponden al `key` de cada campo configurado en `organizations.config.onboarding.profile_fields`
- No se valida el schema en DB — la validación ocurre en el backend contra la config de la org
- `DEFAULT '{}'` para que usuarios existentes tengan un JSON vacío válido
- No se necesita índice GIN porque las queries no filtran por profile_data
