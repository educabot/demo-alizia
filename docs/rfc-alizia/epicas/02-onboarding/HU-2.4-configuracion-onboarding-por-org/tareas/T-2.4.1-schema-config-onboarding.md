## Go structs para parseo

### `src/core/entities/onboarding_config.go`

```go
package entities

// OnboardingConfig representa el bloque "onboarding" del JSONB config de la org
type OnboardingConfig struct {
    SkipAllowed   bool           `json:"skip_allowed"`
    ProfileFields []ProfileField `json:"profile_fields"`
    TourSteps     []TourStepConfig `json:"tour_steps"`
}

type ProfileField struct {
    Key      string   `json:"key"`
    Label    string   `json:"label"`
    Type     string   `json:"type"` // text, number, select, multiselect
    Options  []string `json:"options,omitempty"`
    Required bool     `json:"required"`
}

type TourStepConfig struct {
    Key             string   `json:"key"`
    Title           string   `json:"title"`
    Description     string   `json:"description"`
    Order           int      `json:"order"`
    Roles           []string `json:"roles"`
    RequiresFeature string   `json:"requires_feature,omitempty"`
}
```

## Endpoint

### GET /organizations/:orgId/onboarding-config

```go
// Retorna la config de onboarding de la org
// El frontend la usa para renderizar el formulario dinámico
func (h *OrgHandler) GetOnboardingConfig(ctx context.Context, r *web.Request) web.Response {
    orgID := r.AuthOrgID()
    org, err := h.orgProvider.GetByID(ctx, orgID)
    if err != nil {
        return rest.HandleError(err)
    }

    config := extractOnboardingConfig(org.Config)
    return web.OK(config)
}
```

## Notas

- No requiere migración — el bloque `onboarding` se agrega al JSON existente de `organizations.config`
- El frontend lee `profile_fields` para renderizar el formulario dinámicamente
- El frontend NO lee `tour_steps` directamente — usa el endpoint de T-2.3.1 que filtra por rol y features
- `skip_allowed` lo consulta el frontend para mostrar/ocultar el botón "Saltar"
- Los `profile_fields.type` soportados son: `text`, `number`, `select`, `multiselect`
- `requires_feature` en tour steps es opcional — si no está, el paso siempre se muestra
- La config la carga el equipo de implementación al hacer setup de la org, no el usuario final
