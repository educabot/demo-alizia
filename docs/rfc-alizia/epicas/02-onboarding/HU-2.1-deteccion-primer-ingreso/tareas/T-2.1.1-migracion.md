## Descripción

Agregar columna `onboarding_completed_at` a la tabla `users` para trackear si el usuario completó el flujo de onboarding.

## Migración

### `migrations/XXX_add_onboarding_to_users.up.sql`

```sql
ALTER TABLE users ADD COLUMN onboarding_completed_at TIMESTAMP NULL;

COMMENT ON COLUMN users.onboarding_completed_at IS 'Timestamp when user completed onboarding flow. NULL = not completed.';
```

### `migrations/XXX_add_onboarding_to_users.down.sql`

```sql
ALTER TABLE users DROP COLUMN onboarding_completed_at;
```

## Notas

- `NULL` = onboarding no completado, timestamp = fecha en que completó
- Usar timestamp en vez de boolean permite saber cuándo completó (útil para métricas)
- No se necesita índice — la query siempre filtra por user ID, no por este campo
- Para usuarios existentes al momento del deploy, correr: `UPDATE users SET onboarding_completed_at = NOW() WHERE onboarding_completed_at IS NULL;`
