# T-2.3.1: Endpoint tour steps

**Historia:** HU-2.3 — Product tour
**Tipo:** Backend
**Estado:** ⬜ Pendiente
**Fase:** Post-MVP

---

## Descripción

Endpoint que retorna los pasos del product tour filtrados por los roles del usuario autenticado y los feature flags activos de su organización. Si la org no tiene pasos configurados, retorna un tour default mínimo.

## Usecase

### `src/core/usecases/onboarding/get_tour_steps.go`

```go
package onboarding

import (
    "context"
    "go-alizia/src/core/providers"
)

type TourStep struct {
    Key         string `json:"key"`
    Title       string `json:"title"`
    Description string `json:"description"`
    Order       int    `json:"order"`
}

var defaultTourSteps = []TourStep{
    {Key: "welcome", Title: "Bienvenido a Alizia", Description: "Alizia te ayuda a planificar el año escolar de manera colaborativa.", Order: 1},
    {Key: "explore", Title: "Explorá la plataforma", Description: "Navegá por las secciones para descubrir las herramientas disponibles.", Order: 2},
}

type GetTourStepsUseCase struct {
    orgProvider  providers.OrganizationProvider
    userProvider providers.UserProvider
}

func NewGetTourStepsUseCase(op providers.OrganizationProvider, up providers.UserProvider) *GetTourStepsUseCase {
    return &GetTourStepsUseCase{orgProvider: op, userProvider: up}
}

func (uc *GetTourStepsUseCase) Execute(ctx context.Context, userID uint, orgID uint) ([]TourStep, error) {
    org, err := uc.orgProvider.GetByID(ctx, orgID)
    if err != nil {
        return nil, err
    }

    user, err := uc.userProvider.FindByID(ctx, userID)
    if err != nil {
        return nil, err
    }

    // Extraer config de tour de org.Config
    tourConfig := extractTourConfig(org.Config)
    if len(tourConfig) == 0 {
        return defaultTourSteps, nil
    }

    // Filtrar por roles del usuario
    userRoles := extractRoleNames(user.Roles)

    // Filtrar por feature flags activos de la org
    activeFeatures := extractActiveFeatures(org.Config)

    var steps []TourStep
    for _, step := range tourConfig {
        if matchesRoles(step.Roles, userRoles) && matchesFeature(step.RequiresFeature, activeFeatures) {
            steps = append(steps, TourStep{
                Key:         step.Key,
                Title:       step.Title,
                Description: step.Description,
                Order:       step.Order,
            })
        }
    }

    // Dedup por key (usuario con múltiples roles)
    steps = dedupByKey(steps)
    sortByOrder(steps)

    return steps, nil
}
```

## Endpoint

```go
// GET /users/me/onboarding/tour-steps
// Retorna los pasos del tour filtrados por rol y features
func (h *UserHandler) GetTourSteps(ctx context.Context, r *web.Request) web.Response {
    userID := r.AuthUserID()
    orgID := r.AuthOrgID()

    steps, err := h.getTourStepsUC.Execute(ctx, userID, orgID)
    if err != nil {
        return rest.HandleError(err)
    }
    return web.OK(steps)
}
```

## Notas

- El filtrado es server-side para no exponer config interna al frontend
- El frontend recibe solo los pasos que debe mostrar, en orden
- Dedup por `key` maneja el caso de usuario con múltiples roles sin duplicar pasos compartidos (ej: "welcome")
- Si la org no tiene tour config, retorna 2 pasos default genéricos
- La implementación visual del tour (tooltips, modals, highlights) es responsabilidad del frontend
