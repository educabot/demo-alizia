## Descripcion

Crear la configuracion del AI Provider, los errores tipados de la capa core, y el wiring en el contenedor de dependencias. El provider se inicializa a partir de variables de entorno y se inyecta como `providers.AIProvider` en los usecases que lo necesiten.

### Configuracion

```go
// src/infra/ai/config.go
package ai

import (
    "time"

    bcfg "github.com/educabot/team-ai-toolkit/config"
    "github.com/Azure/azure-sdk-for-go/sdk/ai/azopenai"
    "github.com/Azure/azure-sdk-for-go/sdk/azcore"
)

// AIConfig contiene la configuracion para el AI Provider.
type AIConfig struct {
    Endpoint string        // AZURE_OPENAI_ENDPOINT (requerido)
    APIKey   string        // AZURE_OPENAI_API_KEY (requerido)
    Model    string        // AZURE_OPENAI_MODEL (default: "gpt-5-mini")
    Timeout  time.Duration // AI_TIMEOUT (default: 30s)
}

// NewAIConfig lee la configuracion de variables de entorno.
func NewAIConfig() AIConfig {
    return AIConfig{
        Endpoint: bcfg.MustEnv("AZURE_OPENAI_ENDPOINT"),
        APIKey:   bcfg.MustEnv("AZURE_OPENAI_API_KEY"),
        Model:    bcfg.EnvOr("AZURE_OPENAI_MODEL", "gpt-5-mini"),
        Timeout:  parseDurationOr("AI_TIMEOUT", 30*time.Second),
    }
}

// NewAzureOpenAIProvider crea un provider configurado con las credenciales de Azure.
func NewAzureOpenAIProvider(cfg AIConfig) (*AzureOpenAIProvider, error) {
    keyCredential := azcore.NewKeyCredential(cfg.APIKey)
    client, err := azopenai.NewClientWithKeyCredential(cfg.Endpoint, keyCredential, nil)
    if err != nil {
        return nil, fmt.Errorf("ai: failed to create Azure OpenAI client: %w", err)
    }

    return &AzureOpenAIProvider{
        client:  client,
        model:   cfg.Model,
        timeout: cfg.Timeout,
    }, nil
}

// parseDurationOr lee una variable de entorno como duration, con fallback.
func parseDurationOr(key string, fallback time.Duration) time.Duration {
    val := os.Getenv(key)
    if val == "" {
        return fallback
    }
    d, err := time.ParseDuration(val)
    if err != nil {
        return fallback
    }
    return d
}
```

### Variables de entorno

| Variable | Requerida | Default | Descripcion |
|----------|-----------|---------|-------------|
| `AZURE_OPENAI_ENDPOINT` | Si | — | URL del recurso Azure OpenAI |
| `AZURE_OPENAI_API_KEY` | Si | — | API key del recurso Azure OpenAI |
| `AZURE_OPENAI_MODEL` | No | `gpt-5-mini` | Nombre del deployment del modelo |
| `AI_TIMEOUT` | No | `30s` | Timeout por request al modelo |

## Comportamiento

1. `NewAIConfig()` lee las variables de entorno. Si faltan `ENDPOINT` o `API_KEY`, paniquea via `MustEnv` (fallo rapido en startup)
2. `NewAzureOpenAIProvider()` crea el cliente del SDK de Azure con key credential
3. El provider se inyecta como `providers.AIProvider` (interfaz) en los usecases — nunca como tipo concreto
4. Si el provider falla al inicializar, la app no arranca (`log.Fatalf`)

## Notas

- Se usa `bcfg.MustEnv` y `bcfg.EnvOr` de `team-ai-toolkit/config` para consistencia con el resto de la config
- El provider se inyecta como interfaz para facilitar testing con mocks
- La config de AI es independiente de `config.BaseConfig` pero se puede integrar en el `Config` struct del proyecto (similar a como se muestra en `team-ai-toolkit.md`)
- `parseDurationOr` permite configurar el timeout sin recompilar (util para ajustar en produccion)
