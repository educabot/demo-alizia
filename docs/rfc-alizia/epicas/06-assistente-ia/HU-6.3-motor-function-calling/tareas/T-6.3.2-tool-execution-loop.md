# T-6.3.2: Tool execution loop

**Historia:** HU-6.3 — Motor de function calling
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripcion

Implementar el loop de ejecucion de tools que maneja la conversacion multi-turn entre el LLM y las tools registradas. El loop envia mensajes al modelo, detecta tool calls en la respuesta, ejecuta los handlers, agrega los resultados y reenvia hasta obtener una respuesta final o alcanzar el limite de iteraciones.

## Implementacion

```go
// src/core/usecases/ai/tool_loop.go
package ai

import (
    "context"
    "fmt"
    "log"

    "go-alizia/src/core/providers"
)

const maxToolIterations = 5

// AIClient es la interfaz minima que necesita el loop para comunicarse con el LLM.
// Se satisface con la implementacion de Azure OpenAI (HU-6.1).
type AIClient interface {
    SendCompletion(ctx context.Context, messages []providers.ChatMessage, tools []providers.ToolDefinition) (*CompletionResponse, error)
}

// CompletionResponse es la respuesta cruda del LLM (una sola iteracion).
type CompletionResponse struct {
    Content    string
    ToolCalls  []providers.ToolCall
    TokensUsed int
}

// RunToolLoop maneja el loop completo de function calling.
// Envia mensajes al LLM, ejecuta tools si las hay, y repite hasta obtener
// una respuesta final sin tool calls o alcanzar el limite de iteraciones.
func RunToolLoop(ctx context.Context, client AIClient, messages []providers.ChatMessage, registry *ToolRegistry) (*providers.ChatWithToolsResponse, error) {
    toolsUsed := []string{}
    totalTokens := 0

    for i := 0; i < maxToolIterations; i++ {
        resp, err := client.SendCompletion(ctx, messages, registry.GetDefinitions())
        if err != nil {
            return nil, fmt.Errorf("ai completion error (iteration %d): %w", i, err)
        }

        totalTokens += resp.TokensUsed

        // Si no hay tool calls, terminamos — esta es la respuesta final
        if len(resp.ToolCalls) == 0 {
            return &providers.ChatWithToolsResponse{
                Message:    resp.Content,
                ToolsUsed:  toolsUsed,
                TokensUsed: totalTokens,
            }, nil
        }

        // Agregar el mensaje del assistant (con tool calls) al historial
        messages = append(messages, providers.ChatMessage{
            Role:      "assistant",
            Content:   resp.Content,
            ToolCalls: resp.ToolCalls,
        })

        // Ejecutar cada tool call y agregar los resultados
        for _, tc := range resp.ToolCalls {
            toolsUsed = append(toolsUsed, tc.Function.Name)

            result, err := registry.Execute(tc.Function.Name, tc.Function.Arguments)
            if err != nil {
                // Error NO es fatal — se envia como resultado al LLM
                // para que informe al usuario de forma amigable
                result = fmt.Sprintf("Error ejecutando %s: %s", tc.Function.Name, err.Error())
                log.Printf("tool error: %s: %v", tc.Function.Name, err)
            }

            // Agregar resultado de la tool como mensaje
            messages = append(messages, providers.ChatMessage{
                Role:       "tool",
                Content:    result,
                ToolCallID: tc.ID,
            })
        }
    }

    // Se alcanzo el maximo de iteraciones — retornar warning
    return &providers.ChatWithToolsResponse{
        Message:    "Se alcanzo el maximo de operaciones. Por favor, intenta de nuevo con una instruccion mas especifica.",
        ToolsUsed:  toolsUsed,
        TokensUsed: totalTokens,
    }, nil
}
```

## Flujo

```
Mensaje del usuario
  -> Enviar al LLM con definiciones de tools
  -> Si tool_calls en la respuesta:
      -> Ejecutar cada tool via registry.Execute()
      -> Agregar mensaje assistant + resultados tool al historial
      -> Enviar de nuevo (iteration++)
  -> Si no hay tool_calls -> retornar mensaje del assistant
  -> Si max iteraciones alcanzadas -> retornar warning
```

## Comportamiento

1. El loop recibe los mensajes actuales de la conversacion, el client para comunicarse con el LLM, y el registry con las tools disponibles
2. En cada iteracion, envia los mensajes y las definiciones de tools al LLM
3. Si el LLM responde sin tool calls → la conversacion termino, retorna el mensaje final
4. Si el LLM responde con tool calls → ejecuta cada una, agrega resultados y repite
5. Si se ejecutan multiples tool calls en paralelo (mismo response), todas se ejecutan antes de reenviar
6. Los errores de tools NO son fatales — se envian como resultado para que el LLM responda con un mensaje amigable al usuario
7. Despues de 5 iteraciones sin respuesta final → retorna un mensaje de warning

## Notas

- Este es el motor central — consumido por HU-4.6 (coord doc chat) y cualquier futura feature de chat
- `AIClient` es una interfaz minima para desacoplar el loop de la implementacion concreta de Azure OpenAI
- Los tokens se acumulan a traves de todas las iteraciones para tracking de uso
- El cap de 5 iteraciones previene loops infinitos si el LLM sigue pidiendo tools indefinidamente
- Los errores del LLM (rate limit, timeout, etc.) SI son fatales y se propagan al caller
