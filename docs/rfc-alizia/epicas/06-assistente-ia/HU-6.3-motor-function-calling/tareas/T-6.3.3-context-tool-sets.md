# T-6.3.3: Context-specific tool sets

**Historia:** HU-6.3 — Motor de function calling
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripcion

Definir factory functions que crean tool sets especificos para cada contexto de chat. Cada contexto (documento de coordinacion, planificacion docente, etc.) registra sus propias tools con handlers que delegan en interfaces de dominio via dependency injection.

## Implementacion

```go
// src/core/usecases/ai/tool_sets.go
package ai

import (
    "encoding/json"
    "fmt"

    "go-alizia/src/core/providers"
)

// --- Coord Doc Tool Set (para HU-4.6) ---

// CoordDocUpdater define las operaciones que las tools del chat de coord doc pueden ejecutar.
// Implementado por el usecase de coordinacion o un adapter dedicado.
type CoordDocUpdater interface {
    UpdateSection(key, content string) (string, error)
    AppendToSection(key, content string) (string, error)
    UpdateClassTitle(subjectID, classNumber int, title string) (string, error)
    UpdateClassTopics(subjectID, classNumber int, topicIDs []int) (string, error)
}

// NewCoordDocToolSet crea el registry con tools para el chat del documento de coordinacion.
func NewCoordDocToolSet(updater CoordDocUpdater) *ToolRegistry {
    registry := NewToolRegistry()

    // Tool: update_section
    registry.Register(
        providers.ToolDefinition{
            Name:        "update_section",
            Description: "Reemplaza el contenido completo de una seccion del documento de coordinacion.",
            Parameters: json.RawMessage(`{
                "type": "object",
                "properties": {
                    "section_key": {
                        "type": "string",
                        "description": "Clave de la seccion (ej: problem_edge, methodology, evaluation)"
                    },
                    "content": {
                        "type": "string",
                        "description": "Nuevo contenido completo de la seccion"
                    }
                },
                "required": ["section_key", "content"]
            }`),
        },
        func(args string) (string, error) {
            var p struct {
                SectionKey string `json:"section_key"`
                Content    string `json:"content"`
            }
            if err := json.Unmarshal([]byte(args), &p); err != nil {
                return "", fmt.Errorf("error parseando argumentos: %w", err)
            }
            return updater.UpdateSection(p.SectionKey, p.Content)
        },
    )

    // Tool: append_to_section
    registry.Register(
        providers.ToolDefinition{
            Name:        "append_to_section",
            Description: "Agrega texto al final de una seccion existente, preservando el contenido actual.",
            Parameters: json.RawMessage(`{
                "type": "object",
                "properties": {
                    "section_key": {
                        "type": "string",
                        "description": "Clave de la seccion"
                    },
                    "content": {
                        "type": "string",
                        "description": "Texto a agregar al final"
                    }
                },
                "required": ["section_key", "content"]
            }`),
        },
        func(args string) (string, error) {
            var p struct {
                SectionKey string `json:"section_key"`
                Content    string `json:"content"`
            }
            if err := json.Unmarshal([]byte(args), &p); err != nil {
                return "", fmt.Errorf("error parseando argumentos: %w", err)
            }
            return updater.AppendToSection(p.SectionKey, p.Content)
        },
    )

    // Tool: update_class_title
    registry.Register(
        providers.ToolDefinition{
            Name:        "update_class_title",
            Description: "Cambia el titulo de una clase especifica en el plan de clases de una materia.",
            Parameters: json.RawMessage(`{
                "type": "object",
                "properties": {
                    "subject_id": {
                        "type": "integer",
                        "description": "ID de la materia"
                    },
                    "class_number": {
                        "type": "integer",
                        "description": "Numero de clase (1-based)"
                    },
                    "title": {
                        "type": "string",
                        "description": "Nuevo titulo de la clase"
                    }
                },
                "required": ["subject_id", "class_number", "title"]
            }`),
        },
        func(args string) (string, error) {
            var p struct {
                SubjectID   int    `json:"subject_id"`
                ClassNumber int    `json:"class_number"`
                Title       string `json:"title"`
            }
            if err := json.Unmarshal([]byte(args), &p); err != nil {
                return "", fmt.Errorf("error parseando argumentos: %w", err)
            }
            return updater.UpdateClassTitle(p.SubjectID, p.ClassNumber, p.Title)
        },
    )

    // Tool: update_class_topics
    registry.Register(
        providers.ToolDefinition{
            Name:        "update_class_topics",
            Description: "Actualiza los topics/categorias asignados a una clase especifica.",
            Parameters: json.RawMessage(`{
                "type": "object",
                "properties": {
                    "subject_id": {
                        "type": "integer",
                        "description": "ID de la materia"
                    },
                    "class_number": {
                        "type": "integer",
                        "description": "Numero de clase (1-based)"
                    },
                    "topic_ids": {
                        "type": "array",
                        "items": {"type": "integer"},
                        "description": "IDs de los topics/categorias a asignar"
                    }
                },
                "required": ["subject_id", "class_number", "topic_ids"]
            }`),
        },
        func(args string) (string, error) {
            var p struct {
                SubjectID   int   `json:"subject_id"`
                ClassNumber int   `json:"class_number"`
                TopicIDs    []int `json:"topic_ids"`
            }
            if err := json.Unmarshal([]byte(args), &p); err != nil {
                return "", fmt.Errorf("error parseando argumentos: %w", err)
            }
            return updater.UpdateClassTopics(p.SubjectID, p.ClassNumber, p.TopicIDs)
        },
    )

    return registry
}
```

## Comportamiento

1. Cada contexto de chat tiene su propia factory function (`NewCoordDocToolSet`, y en el futuro `NewLessonPlanToolSet`, etc.)
2. La factory recibe una interfaz de dominio (`CoordDocUpdater`) que encapsula las operaciones permitidas
3. Se crea un `ToolRegistry` nuevo y se registran las tools especificas del contexto
4. Los handlers parsean los argumentos JSON y delegan en la interfaz de dominio
5. El registry resultante se pasa al `RunToolLoop` (T-6.3.2) para la conversacion

## Patron de extension

Para agregar un nuevo contexto de chat (ej: lesson plan):

```go
// 1. Definir la interfaz de dominio
type LessonPlanUpdater interface {
    UpdateObjective(planID int, objective string) (string, error)
    AddActivity(planID int, activity Activity) (string, error)
}

// 2. Crear la factory
func NewLessonPlanToolSet(updater LessonPlanUpdater) *ToolRegistry {
    registry := NewToolRegistry()
    // ... registrar tools especificas
    return registry
}
```

## Notas

- Los tool sets se crean **por request** — capturan el documento/plan especifico que se esta editando via la interfaz inyectada
- La interfaz `CoordDocUpdater` permite mockear en tests sin tocar la base de datos
- Los parametros de cada tool siguen el formato JSON Schema que espera OpenAI
- Los errores de parsing de argumentos se propagan al LLM como tool result (via el manejo de errores del loop)
- Futuros contextos (lesson plan chat, content generation) siguen el mismo patron: interfaz + factory
