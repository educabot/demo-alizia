# T-6.4.1: Context builder coordinacion

**Historia:** HU-6.4 — Contexto y enriquecimiento
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripcion

Implementar el context builder para documentos de coordinacion que transforma la entidad `CoordinationDocument` en texto enriquecido con formato markdown, optimizado para consumo del LLM. El builder incluye area, periodo, topics seleccionados, materias con sus topics y cantidad de clases, y secciones existentes del documento.

## Implementacion

```go
// src/core/usecases/ai/context_coord_doc.go
package ai

import (
    "fmt"
    "strings"

    "go-alizia/src/core/entities"
)

// BuildCoordDocContext creates a rich text context from a coordination document
// for use in AI prompt generation (sections, class plans).
func BuildCoordDocContext(doc *entities.CoordinationDocument) string {
    var parts []string

    // Area
    if doc.Area != nil {
        parts = append(parts, fmt.Sprintf("## Area\n%s", doc.Area.Name))
    }

    // Periodo
    parts = append(parts, fmt.Sprintf("## Periodo\n%s a %s",
        doc.StartDate.Format("02/01/2006"),
        doc.EndDate.Format("02/01/2006")))

    // Topics seleccionados
    if len(doc.Topics) > 0 {
        topicNames := make([]string, len(doc.Topics))
        for i, t := range doc.Topics {
            topicNames[i] = t.Name
        }
        parts = append(parts, fmt.Sprintf("## Topics seleccionados\n%s", strings.Join(topicNames, ", ")))
    }

    // Materias con sus topics y cantidad de clases
    if len(doc.Subjects) > 0 {
        var subjectLines []string
        for _, s := range doc.Subjects {
            topicNames := make([]string, len(s.Topics))
            for i, t := range s.Topics {
                topicNames[i] = t.Name
            }
            subjectLines = append(subjectLines,
                fmt.Sprintf("- **%s** (%d clases): %s", s.Subject.Name, s.ClassCount, strings.Join(topicNames, ", ")))
        }
        parts = append(parts, fmt.Sprintf("## Materias\n%s", strings.Join(subjectLines, "\n")))
    }

    // Secciones existentes (si las hay)
    if doc.Sections != nil {
        sections := doc.ParseSections()
        if len(sections) > 0 {
            var sectionLines []string
            for key, val := range sections {
                if v, ok := val["value"].(string); ok && v != "" {
                    sectionLines = append(sectionLines, fmt.Sprintf("### %s\n%s", key, v))
                }
            }
            if len(sectionLines) > 0 {
                parts = append(parts, fmt.Sprintf("## Secciones del documento\n%s", strings.Join(sectionLines, "\n\n")))
            }
        }
    }

    return strings.Join(parts, "\n\n")
}
```

## Comportamiento

1. Recibe un `*entities.CoordinationDocument` con datos pre-cargados (area, topics, subjects, sections)
2. Construye un string con formato markdown, usando headers `##` para cada bloque
3. Omite bloques cuyo campo esta vacio o nil (ej: si no hay topics, no aparece la seccion "Topics seleccionados")
4. Las secciones existentes se iteran y solo se incluyen las que tienen valor no vacio
5. El resultado se pasa a `SystemPromptBuilder.AddContext("Documento", contexto)` para armar el system prompt

## Notas

- Este builder es usado por HU-4.3 (GenerateSections), HU-4.4 (GenerateClassPlan) y HU-4.6 (Chat system prompt)
- El output es markdown formateado para claridad del LLM
- Campos vacios/nil se omiten para evitar ruido
- Los topics se listan por nombre (no IDs) ya que eso es lo que el LLM necesita
