# T-1.3.4: Tests de autorización por rol

**Historia:** HU-1.3 — Middleware de autorización
**Tipo:** Testing
**Estado:** ⬜ Pendiente

---

## Descripción

Tests unitarios para el middleware `RequireRole` y tests de integración para la cadena completa de interceptors.

## Tests unitarios — RequireRole

```go
// internal/middleware/require_role_test.go

func TestRequireRole_AllowedSingleRole(t *testing.T) {
    // User con rol "coordinator", endpoint requiere "coordinator" → pasa
}

func TestRequireRole_DeniedSingleRole(t *testing.T) {
    // User con rol "teacher", endpoint requiere "coordinator" → 403
}

func TestRequireRole_AllowedMultipleRoles(t *testing.T) {
    // User con roles ["teacher", "coordinator"], endpoint requiere "coordinator" → pasa
}

func TestRequireRole_AllowedAnyOfRequired(t *testing.T) {
    // User con rol "admin", endpoint requiere "coordinator" o "admin" → pasa
}

func TestRequireRole_NoClaims(t *testing.T) {
    // Context sin claims → 403
}

func TestRequireRole_EmptyRoles(t *testing.T) {
    // Claims presentes pero Roles vacío → 403
}
```

## Tests de integración — Cadena completa

```go
// internal/middleware/chain_test.go

func TestChain_ValidTokenCorrectRole(t *testing.T) {
    // JWT válido + rol correcto → 200
}

func TestChain_ValidTokenWrongRole(t *testing.T) {
    // JWT válido + rol incorrecto → 403
}

func TestChain_NoToken(t *testing.T) {
    // Sin Authorization header → 401 (falla en auth, nunca llega a role check)
}

func TestChain_InvalidToken(t *testing.T) {
    // JWT expirado → 401 (falla en auth)
}
```

## Notas

- Para los tests unitarios de RequireRole, inyectar claims manualmente en el context
- Para los tests de integración, usar `httptest` con la cadena completa de middleware
- Usar JWTs de test firmados con una key conocida (no depender de servicios externos)
- Los tests de integración validan el orden: primero 401, luego 403
