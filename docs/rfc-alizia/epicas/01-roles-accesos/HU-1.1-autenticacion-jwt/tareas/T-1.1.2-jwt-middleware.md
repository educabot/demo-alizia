# T-1.1.2: Integrar JWT middleware (JWKS)

**Historia:** HU-1.1 — Autenticación JWT
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Integrar el middleware de autenticación JWT de `team-ai-toolkit/tokens` para validar tokens via JWKS.

## Implementación

```go
// cmd/main.go o cmd/routes.go
authMiddleware := tokens.NewAuthInterceptor(tokens.AuthConfig{
    Domain:   cfg.JWKSDomain,
    Audience: cfg.JWKSAudience,
})

// Aplicar a todas las rutas protegidas
api := engine.Group("/api/v1")
api.Use(web.AdaptMiddleware(authMiddleware))
```

## Comportamiento

1. Extrae Bearer token del header `Authorization`
2. Valida firma via JWKS endpoint (`https://{domain}/.well-known/jwks.json`)
3. Valida `aud` (audience) y `iss` (issuer)
4. Parsea claims: `user_id`, `org_id`, `roles`, `email`, `name`
5. Inyecta claims en el contexto via `tokens.SetClaims()`

## Errores

- Sin header Authorization → 401 `{"error": "missing_token"}`
- Token malformado o expirado → 401 `{"error": "invalid_token"}`
- Audience no coincide → 401 `{"error": "invalid_token"}`

## Notas

- team-ai-toolkit/tokens valida JWT via JWKS
- JWKS se cachea (normalmente 24h) para no hacer HTTP en cada request
- Si team-ai-toolkit ya soporta JWKS, solo configurar. Si no, actualizar la librería primero
