# T-1.1.1: Configurar JWT authentication

**Historia:** HU-1.1 — Autenticación JWT
**Tipo:** Infra
**Estado:** ⬜ Pendiente

---

## Descripción

Configurar la autenticación JWT para Alizia via team-ai-toolkit/tokens.

## Pasos

1. Configurar JWKS endpoint para validación de tokens:
   - Audience: `https://api.alizia.educabot.com` (o el que defina el equipo)
   - Signing Algorithm: RS256
2. Configurar custom claims en el JWT:
   - `org_id` — organization ID del usuario
   - `roles` — array de roles del usuario (teacher, coordinator, admin)
3. Obtener JWKS domain y audience para staging

## Variables de entorno resultantes

```
JWKS_DOMAIN=<jwks-domain-staging>
JWKS_AUDIENCE=https://api.alizia.educabot.com
```

## Notas

- El custom claim de `org_id` es CRITICO para multi-tenancy — viene del JWT, no del request
- Los roles tambien vienen del JWT como claim custom
- El frontend obtiene el token y lo envia como Bearer token
- team-ai-toolkit/tokens valida via JWKS endpoint
