# T-1.1.5: Tests de autenticación

**Historia:** HU-1.1 — Autenticación JWT
**Tipo:** Test
**Estado:** ⬜ Pendiente

---

## Descripción

Escribir tests que validen el flujo completo de autenticación y tenant isolation.

## Test cases

| # | Caso | Input | Expected |
|---|------|-------|----------|
| 1 | Request sin Authorization header | GET /api/v1/areas (sin header) | 401 `{"error": "missing_token"}` |
| 2 | Request con JWT inválido | GET /api/v1/areas + Bearer `invalid` | 401 `{"error": "invalid_token"}` |
| 3 | Request con JWT expirado | GET /api/v1/areas + Bearer expired_token | 401 `{"error": "invalid_token"}` |
| 4 | Request con JWT válido | GET /api/v1/areas + Bearer valid_token | 200 + claims accesibles en context |
| 5 | JWT de org A no ve datos de org B | Crear área en org A, GET con JWT de org B | [] vacío |

## Approach

- Generar JWTs de test con una key RSA local (no depender de servicios externos en tests)
- Mockear el JWKS endpoint o usar `tokens.NewAuthInterceptor` con key fija para tests
- Usar `web.MockRequest` de team-ai-toolkit para tests unitarios del middleware
- Tests de integración con HTTP real para el flujo completo

## Archivos

- `internal/admin/handlers/area_handler_test.go` (o donde estén los primeros endpoints)
- `cmd/auth_test.go` (test de integración del middleware chain)
