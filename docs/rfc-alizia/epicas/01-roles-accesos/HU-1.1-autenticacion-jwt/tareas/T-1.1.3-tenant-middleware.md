# T-1.1.3: Integrar tenant middleware

**Historia:** HU-1.1 — Autenticación JWT
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Integrar el middleware de tenant de `team-ai-toolkit/tokens` que extrae `org_id` del JWT y lo inyecta en el contexto para filtrado automático.

## Implementación

```go
// Se aplica DESPUÉS del auth middleware
tenantMiddleware := tokens.NewTenantInterceptor()

api := engine.Group("/api/v1")
api.Use(web.AdaptMiddleware(authMiddleware))
api.Use(web.AdaptMiddleware(tenantMiddleware))
```

## Comportamiento

1. Lee claims del contexto (ya inyectados por auth middleware)
2. Extrae `org_id` de los claims
3. Lo inyecta como valor accesible via `tokens.OrgID(ctx)`
4. Si no hay `org_id` en los claims → 401

## Uso en handlers/repositories

```go
// En cualquier handler o repository
orgID := tokens.OrgID(r.Context())

// En GORM queries
db.Where("organization_id = ?", orgID).Find(&areas)
```

## Notas

- TODAS las queries que tocan datos de tenant deben filtrar por `org_id`
- Nunca confiar en `org_id` del request body/params — siempre del JWT
- Riesgo mitigado: test T1-T4 validan isolation entre orgs
