# T-1.1.4: Config: JWT auth env vars

**Historia:** HU-1.1 — Autenticación JWT
**Tipo:** Backend
**Estado:** ⬜ Pendiente

---

## Descripción

Agregar las variables de configuración de JWT auth al config struct de la app.

## Implementación

```go
// config/config.go
type Config struct {
    config.BaseConfig // de team-ai-toolkit

    JWKSDomain   string
    JWKSAudience string
}

func Load() Config {
    base := config.LoadBase()
    return Config{
        BaseConfig:   base,
        JWKSDomain:   config.MustEnv("JWKS_DOMAIN"),
        JWKSAudience: config.MustEnv("JWKS_AUDIENCE"),
    }
}
```

## Variables de entorno

| Variable | Descripción | Ejemplo |
|----------|-------------|---------|
| `JWKS_DOMAIN` | JWKS domain for JWT validation | `<jwks-domain-staging>` |
| `JWKS_AUDIENCE` | JWT API audience identifier | `https://api.alizia.educabot.com` |

## Notas

- `MustEnv` panics si la variable no existe — fail fast al startup
- `BaseConfig` ya trae: `PORT`, `DATABASE_URL`, `ENV`, `BUGSNAG_API_KEY`
