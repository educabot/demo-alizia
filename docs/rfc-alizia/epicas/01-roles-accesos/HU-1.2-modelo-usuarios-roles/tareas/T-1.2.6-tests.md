# T-1.2.6: Tests del repository

**Historia:** HU-1.2 — Modelo de usuarios y roles
**Tipo:** Test
**Estado:** ⬜ Pendiente

---

## Descripción

Tests de integración para los repositories de Organization y User usando una DB de test real (no mocks).

## Test cases

### OrganizationRepository

| # | Caso | Expected |
|---|------|----------|
| 1 | FindByID existente | Retorna org con config JSONB parseado |
| 2 | FindByID inexistente | Retorna ErrNotFound |
| 3 | FindBySlug existente | Retorna org |
| 4 | FindBySlug inexistente | Retorna ErrNotFound |

### UserRepository

| # | Caso | Expected |
|---|------|----------|
| 1 | Create user | User creado con ID asignado |
| 2 | Create user con email duplicado en misma org | Error unique constraint |
| 3 | Create user con mismo email en otra org | OK (permite) |
| 4 | FindByID con Preload Roles | User con roles cargados |
| 5 | FindByOrgID | Solo users de esa org |
| 6 | AssignRole | UserRole creado |
| 7 | AssignRole duplicado | Error unique constraint |
| 8 | RemoveRole | UserRole eliminado |
| 9 | HasRole true | Retorna true |
| 10 | HasRole false | Retorna false |

## Approach

- Usar una DB PostgreSQL de test (docker-compose o CI service)
- Cada test corre migraciones y limpia datos (transaction rollback o truncate)
- `testify/assert` y `testify/require` para assertions
