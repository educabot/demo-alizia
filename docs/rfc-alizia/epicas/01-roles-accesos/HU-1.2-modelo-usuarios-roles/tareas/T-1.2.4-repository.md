## Descripción

Implementar los repositories con GORM para organizations y users.

## Archivos

### `internal/admin/repositories/organization.go`

```go
type organizationRepo struct {
    db *gorm.DB
}

func NewOrganizationRepository(db *gorm.DB) providers.OrganizationRepository {
    return &organizationRepo{db: db}
}

func (r *organizationRepo) FindByID(ctx context.Context, id uint) (*entities.Organization, error) {
    var org entities.Organization
    if err := r.db.WithContext(ctx).First(&org, id).Error; err != nil {
        if errors.Is(err, gorm.ErrRecordNotFound) {
            return nil, apperrors.ErrNotFound
        }
        return nil, err
    }
    return &org, nil
}
```

```go
// FindByID con Preload de roles
func (r *userRepo) FindByID(ctx context.Context, id uint) (*entities.User, error) {
    var user entities.User
    if err := r.db.WithContext(ctx).Preload("Roles").First(&user, id).Error; err != nil {
        // ...
    }
    return &user, nil
}

// FindByOrgID filtra por tenant
func (r *userRepo) FindByOrgID(ctx context.Context, orgID uint) ([]entities.User, error) {
    var users []entities.User
    err := r.db.WithContext(ctx).
        Where("organization_id = ?", orgID).
        Preload("Roles").
        Find(&users).Error
    return users, err
}
```

## Notas

- Siempre filtrar por `organization_id` en queries de tenant
- Usar `Preload("Roles")` para cargar roles del usuario
- Mapear `gorm.ErrRecordNotFound` a `apperrors.ErrNotFound` (de team-ai-toolkit/errors)
- El package `repositories/` ya implica la implementación concreta — no hace falta sufijo `_postgres` en los archivos
