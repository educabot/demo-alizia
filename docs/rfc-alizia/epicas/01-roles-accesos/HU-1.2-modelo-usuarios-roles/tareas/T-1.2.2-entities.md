## Descripción

Crear las entities Go que representan el modelo de datos de organizations, users y roles.

## Archivos

### `internal/admin/entities/organization.go`

```go
type Organization struct {
    ID        uint           `json:"id" gorm:"primaryKey"`
    Name      string         `json:"name"`
    Slug      string         `json:"slug" gorm:"uniqueIndex"`
    Config    datatypes.JSON `json:"config"`
    CreatedAt time.Time      `json:"created_at"`
}
```

### `internal/admin/entities/user.go`

```go
type User struct {
    ID             uint       `json:"id" gorm:"primaryKey"`
    OrganizationID uint       `json:"organization_id"`
    Email          string     `json:"email"`
    Name           string     `json:"name"`
    PasswordHash   *string    `json:"-"` // nullable, no se expone
    AvatarURL      *string    `json:"avatar_url"`
    CreatedAt      time.Time  `json:"created_at"`
    Roles          []UserRole `json:"roles" gorm:"foreignKey:UserID"`
}

type UserRole struct {
    ID     uint   `json:"id" gorm:"primaryKey"`
    UserID uint   `json:"user_id"`
    Role   string `json:"role"` // member_role enum
}
```

## Notas

- `datatypes.JSON` de `gorm.io/datatypes` para el campo JSONB
- `PasswordHash` es pointer (nullable) porque la auth es externa (JWT via team-ai-toolkit/tokens)
- Roles se cargan con Preload cuando se necesita
- No usar `gorm.Model` (tiene campos extra que no necesitamos)
