# T-1.2.1: Migración: organizations + users + user_roles

**Historia:** HU-1.2 — Modelo de usuarios y roles
**Tipo:** Backend / DB
**Estado:** ⬜ Pendiente

---

## Descripción

Crear la primera migración con las tablas base de autenticación y multi-tenancy.

## SQL (up)

```sql
-- Enums
CREATE TYPE member_role AS ENUM ('teacher', 'coordinator', 'admin');

-- Organizations (tenant)
CREATE TABLE organizations (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    slug VARCHAR(100) NOT NULL UNIQUE,
    config JSONB NOT NULL DEFAULT '{}',
    created_at TIMESTAMP NOT NULL DEFAULT NOW()
);

-- Users
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    organization_id INTEGER NOT NULL REFERENCES organizations(id),
    email VARCHAR(255) NOT NULL,
    name VARCHAR(255) NOT NULL,
    password_hash TEXT,
    avatar_url TEXT,
    created_at TIMESTAMP NOT NULL DEFAULT NOW(),
    UNIQUE(email, organization_id)
);

-- User roles (multi-role)
CREATE TABLE user_roles (
    id SERIAL PRIMARY KEY,
    user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    role member_role NOT NULL,
    UNIQUE(user_id, role)
);

-- Indexes
CREATE INDEX idx_users_organization_id ON users(organization_id);
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_user_roles_user_id ON user_roles(user_id);
```

## SQL (down)

```sql
DROP TABLE IF EXISTS user_roles;
DROP TABLE IF EXISTS users;
DROP TABLE IF EXISTS organizations;
DROP TYPE IF EXISTS member_role;
```

## Notas

- `password_hash` es nullable porque en el MVP la auth es via JWT (team-ai-toolkit/tokens, no guardamos passwords localmente)
- `UNIQUE(email, organization_id)` permite mismo email en distintas orgs
- Esta migración se extiende en la misma fase con areas, subjects, topics, etc. (ver Épica 3)
- El campo `config` JSONB de organizations se define con valores por defecto sensatos
