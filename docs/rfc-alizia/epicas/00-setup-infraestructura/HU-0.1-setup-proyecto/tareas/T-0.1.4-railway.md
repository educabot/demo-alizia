# T-0.1.4: Provisionar Railway + PostgreSQL

**Historia:** HU-0.1 — Setup del proyecto
**Tipo:** Infra
**Estado:** ⬜ Pendiente

---

## Descripción

Crear proyecto en Railway con servicio Go + base de datos PostgreSQL.

## Pasos

1. Crear proyecto `alizia-api` en Railway
2. Agregar servicio desde GitHub repo (auto-deploy desde `main`)
3. Agregar plugin PostgreSQL
4. Configurar variables de entorno:
   - `DATABASE_URL` — auto-generada por Railway
   - `PORT` — auto-generada por Railway
   - `JWKS_DOMAIN` — (pendiente HU-1.1)
   - `JWKS_AUDIENCE` — (pendiente HU-1.1)
   - `AZURE_OPENAI_ENDPOINT` — (para Fase 4)
   - `AZURE_OPENAI_API_KEY` — (para Fase 4)
   - `BUGSNAG_API_KEY` — (para error tracking)
   - `ENV` — staging / production
5. Configurar healthcheck: `/health` cada 30s

## Entornos

- **Staging**: auto-deploy desde `main`
- **Production**: deploy manual o desde tag

## Notas

- Railway auto-detecta Dockerfile o Go buildpack
- PostgreSQL 16 recomendado
- Zero vendor lock-in: Dockerfile portable a Render/Fly.io/VPS
